. for user easibily, it runs a local web server on localhost:8080 that interact with backend via API requests. 1st Phase File Upload & Key Generation client side streams file as a raw binary file to the server. server encrypt the file chunk-by-chunk, commits it to disk, server generates & return an cryptographic key to client as a downloadable attachment. Request Constraints """ POST /putFile HTTP/1.1 Host: localhost:8080 Content-Type: application/octet-stream X-Original-Filename: secretFile.ext [RAW BINARY DATA STREAM OVER WEB SOCKET CONNECTIONS] """ where,
Request Line POST REST API, which tells the server that the client is sending data to be processed or stored.
/putFile API route on the server that handles the upload logic.
HTTP/1.1 version of HTTP protocol being used.
Headers Host: localhost:8080 Specifies the domain and port of the server receiving the request
Content-Type: application/octet-stream it tells the server that the body of the request is arbitrary binary data.
It signals to the backend that it shouldn't try to parse the body as JSON, XML,
but rather just read it as a stream of bytes.
X-Original-Filename: secretFile.ext tell the server what the file should be named once it is saved.
Body [RAW BINARY DATA STREAM] raw binary data streaming over web socket connection
Response Constraints """ HTTP/1.1 200 OK Content-Type: application/octet-stream Content-Disposition: attachment; filename="secretFile.ext.key" data(key)... """ where,
Status Line HTTP/1.1 version of HTTP protocol being used.
200 standard HTTP status code for "Success."
OK human-readable explanation of the 200 status code.
Headers Content-Type: application/octet-stream tells the receiving client that the body of the response contains arbitrary binary data.
Content-Disposition: attachment; filename="secretFile.ext.key" tell the browser to save it their disk with given file name.
Response Body data(key)... raw content of file being downloaded as Base64-encoded string
2nd Phase Key Upload & File Retrieval client submit the key as based-64 encoded string to server. server validates the signature, then locates, decrypt, assemble the associated chunks to original file. server return original file as raw binary stream back to client.
